## פרק "קצת עליך" (דמוגרפיה) — מוצע

כל השאלות רשות. עיקרון: טווחים ולא ערכים מדויקים — ביישוב קטן, שילוב של גיל מדויק + שכונה + הרכב משפחה יכול לזהות אדם.

1. **גיל** — טווחים: עד 25 / 26–35 / 36–45 / 46–60 / 61+
2. **מגדר** — אישה / גבר / מעדיף/ה לא לציין
3. **ותק ביישוב** — עד שנתיים / 3–5 / 6–10 / 11–15 / מעל 15 (מחליף את שאלת הטקסט החופשי מהמקור)
4. **סוג מגורים** — בית קבע / קראוון מרכז היישוב / קראוון 468 גבעת החוד (מזין את דילוג פרק הקראוונים)
5. **ילדים בבית** (בחירה מרובה) — גיל רך / יסודי / נוער ז'–י"ב / אין ילדים בבית (מזין דילוג: חוגים, נוער, צופים, משפחות מאמצות, גיל הרך, יסודי)
6. **חברות באגודה** — כן / לא / לא יודע (מהמקור)

בונוס: התשובות כאן מפעילות את הדילוג החכם — במקום לשאול בכל פרק "האם יש לך ילדים בגיל X?", הפרופיל קובע אילו פרקים מוצגים כרלוונטיים.

**כלל פרטיות לדוח הציבורי**: פילוח מוצג רק כשיש לפחות 5 תשובות בפלח (מניעת זיהוי עקיף).

## מודל עצירה והמשך — "פרק = יחידת הגשה"

- בכניסה ראשונה נוצר לתושב מזהה אנונימי (נשמר ב-localStorage).
- **כל פרק נשלח לשרת בנפרד ברגע שהתושב מסיים אותו** — אין "כפתור הגשה סופי" שאם לא הגעת אליו הכול אבד.
- מסך הפרקים מציג סטטוס: ✓ הושלם / ◐ בתהליך / ○ טרם נענה. חוזרים מתי שרוצים וממשיכים.
- תשובות באמצע פרק נשמרות ב-localStorage כטיוטה עד שליחת הפרק.
- **המשך ממכשיר אחר**: לתושב מוצג קוד המשך קצר (המזהה האנונימי); הקלדתו במכשיר אחר מקשרת לאותן תשובות.
- יתרון גדול על גוגל פורמס: גם מי שענה על 5 פרקים ונטש — התשובות שלו נשמרו.
- מבנה נתונים: שורה ב"תשובות" לכל (מזהה, פרק); הדשבורד מאחד לפי מזהה.

## החלטת תשתית

### ההשוואה

**אופציה A — Google Sheets + Apps Script + GitHub Pages** (סטטי, ללא שרת)
- ✅ חינם לחלוטין, בלי חשבונות חדשים — הצוות כבר חי בגוגל
- ✅ התשובות נופלות ישר ל-Google Sheet → שיתוף תוצאות = שיתוף גיליון, כולל Pivot/גרפים למי שרוצה
- ✅ השאלות עצמן יושבות בטאב בגיליון → כל חבר צוות עורך שאלות בלי קוד, בלי דיפלוי
- ✅ בקנה מידה של יישוב (מאות תשובות) — רחוק מאוד ממגבלות ה-quota של Apps Script
- ⚠️ השהיית שרת ~1–2 שניות לשליחה (פותרים עם UX: שליחה ברקע + שמירת טיוטה מקומית)
- ⚠️ אין אימות אמיתי — מתאים להחלטה "פתוח לכולם"; בקרת גישה עתידית תמומש כקוד-כניסה שנבדק ב-Apps Script

**אופציה B — Supabase/Firebase**
- ✅ מסד נתונים אמיתי, מהיר יותר, אימות מובנה
- ❌ עוד חשבון/פלטפורמה לנהל; שיתוף תוצאות עם לא-טכניים דורש לבנות הכול בדשבורד
- ❌ עריכת שאלות מצריכה ממשק אדמין ייעודי או עריכת JSON
- ❌ שכבת החינם תלויה במדיניות הספק

### ההמלצה: אופציה A

הדרישות המרכזיות — עריכה משותפת של שאלות, צפייה משותפת בתוצאות, אפס עלות, אפס תחזוקה — הן בדיוק החוזקות של Sheets. החולשות של Sheets (סקייל, אימות כבד) לא רלוונטיות לפרויקט הזה.

## ארכיטקטורה מתוכננת

```
Google Sheet אחד (משותף לצוות):
├── טאב "שאלות"   — מקור האמת לשאלות: פרק, סוג, טקסט, אפשרויות, לוגיקת דילוג, פעיל/כבוי
├── טאב "תשובות"  — שורה לכל הגשה (timestamp, מזהה אנונימי, תשובות)
├── טאב "הגדרות"  — פתוח/סגור, הודעות, דגל בקרת-גישה עתידי
└── Apps Script Web App — GET (שליפת שאלות+הגדרות) / POST (שמירת הגשה)

אתר סטטי (GitHub Pages):
├── index.html    — הסקר: מובייל-פירסט, RTL, פרק-אחר-פרק, פס התקדמות,
│                    שמירת טיוטה ב-localStorage, לוגיקת דילוג, נגישות (WCAG)
├── dashboard.html — דשבורד צוות (מוגן בקוד): גרפים לפי פרק, ממוצעים, תשובות פתוחות
└── report.html    — דוח ציבורי לתושבים: נתונים מסוננים שהצוות אישר לפרסום
```

### עקרונות UX לסקר (נגזר מגודל הסקר — 246 שאלות!)
1. פרק אחד במסך — ניווט בין פרקים, לא גלילה אינסופית
2. שמירת טיוטה אוטומטית — אפשר לצאת ולחזור
3. לוגיקת דילוג אמיתית — תושב שלא גר בקראוון לא רואה את פרק הקראוונים
4. כל הפרקים רשות — אפשר לדלג ולהגיש חלקית (כמו במקור)
5. סולמות 1–10 כרכיב מגע נוח במובייל, עם תוויות הקצה מהמקור
