# סקר התושבים השנתי — נופי פרת 🌿

מערכת סקר מלאה במקום Google Forms: מובייל-פירסט, RTL, אנונימית (שם רשות), עם הגשה פר-פרק, קוד המשך בין מכשירים, הגרלה למסיימים, דשבורד צוות ודוח ציבורי.

**התשתית**: אתר סטטי (GitHub Pages, חינם) + Google Apps Script + Google Sheets כמסד נתונים.
העלות: **0 ₪ · אפס תחזוקה · הצוות עורך שאלות וצופה בתוצאות ישירות בגיליון.**

## הקבצים

| קובץ | תפקיד |
|---|---|
| `index.html` | הסקר עצמו (מה שהתושבים רואים) |
| `dashboard.html` | דשבורד צוות — מוגן סיסמה |
| `report.html` | דוח ציבורי לתושבים (מצרפי בלבד, נפתח רק כשמחליטים) |
| `setup.html` | דף פנימי — סנכרון השאלות מהקובץ אל הגיליון |
| `survey-data.js` | כל הפרקים והשאלות (מקור לזריעה + fallback מקומי) |
| `appscript.gs` | ה-Backend — מודבק ב-Apps Script |
| `config.js` | הגדרות — כאן מדביקים את כתובת השרת |
| `styles.css`, `utils.js` | עיצוב ועזרים |

## התקנה — 5 שלבים, ~15 דקות

### שלב 1: יצירת הגיליון
1. היכנסו ל-[sheets.new](https://sheets.new) עם חשבון הגוגל של היישוב וצרו גיליון חדש (למשל "סקר תושבים 2026").
2. העתיקו את ה-**ID** של הגיליון — המחרוזת הארוכה בכתובת, בין `/d/` ל-`/edit`:
   `https://docs.google.com/spreadsheets/d/`**`זה-ה-ID`**`/edit`

### שלב 2: Apps Script
1. בגיליון: **Extensions → Apps Script**.
2. מחקו את התוכן שבעורך והדביקו את כל הקובץ `appscript.gs`.
3. בשורה `const SHEET_ID = '...'` הדביקו את ה-ID מהשלב הקודם. שמרו (Ctrl+S).
4. **Deploy → New deployment → ⚙ → Web app**:
   - Execute as: **Me**
   - Who has access: **Anyone**
5. אשרו את ההרשאות (Google יזהיר — זה בסדר, זה הסקריפט שלכם), והעתיקו את כתובת ה-**Web app**.

### שלב 3: חיבור ה-Frontend
1. פתחו את `config.js` והדביקו:
   - `SCRIPT_URL` — הכתובת מהשלב הקודם.
   - `SHEETS_URL` — הכתובת המלאה של הגיליון (לכפתור בדשבורד).

### שלב 4: העלאה ל-GitHub Pages
1. צרו ריפו חדש ב-GitHub והעלו את כל הקבצים (בלי `appscript.gs` — הוא כבר בענן של גוגל, אבל לא מזיק אם עולה).
2. **Settings → Pages → Source: main branch → Save**.
3. תוך דקה יהיה לכם קישור: `https://<שם>.github.io/<ריפו>/`

### שלב 5: זריעת השאלות
1. פתחו `https://<האתר>/setup.html`.
2. הזינו את סיסמת הצוות (ברירת מחדל: `nofim2026` — **החליפו אותה מיד** בטאב "הגדרות" בגיליון!).
3. לחצו "סנכרון שאלות לגיליון" — הטאבים "פרקים" ו"שאלות" יתמלאו.
4. פתחו את האתר הראשי וודאו שהסקר נטען. זהו! 🎉

## ⚠️ שני דברים שחייבים לזכור

1. **כל עריכה של `appscript.gs` מחייבת פרסום מחדש**: Deploy → Manage deployments → ✏ → Version: **New version** → Deploy. בלי זה הקוד החדש לא באוויר.
2. **מומלץ טריגר חימום** (מונע המתנה של 3–5 שניות לגולש הראשון): ב-Apps Script → Triggers (⏰) → Add Trigger → פונקציה `keepWarm` → Time-driven → Every 10 minutes.

## ניהול שוטף — בלי קוד, ישירות בגיליון

| מה רוצים | איפה |
|---|---|
| לתקן נוסח שאלה / להוסיף / להשבית | טאב **שאלות** (עמודת "פעיל" = לא → השאלה נעלמת) |
| להשבית פרק שלם | טאב **פרקים** → "פעיל" = לא |
| לסגור את הסקר למענה | טאב **הגדרות** → `surveyOpen` = לא |
| לפרסם את הדוח הציבורי | טאב **הגדרות** → `publicReport` = כן |
| להחליף סיסמת דשבורד | טאב **הגדרות** → `dashboardPassword` |
| לראות תוצאות בצורה טבלאית | טאב **תוצאות** — שורה לכל עונה, עמודה לכל שאלה |
| לראות את נרשמי ההגרלה | טאב **הגרלה** (שם + טלפון בלבד — בלי קישור לתשובות) |
| שיתוף עם חברי צוות | Share על הגיליון + לתת להם את סיסמת הדשבורד |

## פרטיות ואנונימיות — איך זה בנוי

- לכל עונה נוצר **קוד אנונימי אקראי** (8 תווים) שנשמר רק בדפדפן שלו. אין איסוף של טלפון/מייל/IP.
- שדה השם בסקר הוא **רשות**.
- "אמשיך אחר־כך" ממכשיר אחר: המשתמש משאיר טלפון, אבל בגיליון נשמר **hash בלבד** (טאב "קודי המשך") — אי אפשר לשחזר ממנו את המספר, והצוות לא יכול לקשר טלפון לתשובות.
- ההרשמה להגרלה שומרת **שם וטלפון בלבד, בטאב נפרד, בלי קוד העונה** — אי אפשר לקשר בין הרשמה לתשובות. האימות שהעונה השלים הכול קורה ברגע ההרשמה ולא נשמר.
- הדוח הציבורי מציג **מצרפים בלבד**: בלי טקסט חופשי, בלי שמות, ושאלות עם פחות מ-5 תשובות מוסתרות.
- בקרת גישה עתידית (קוד למשק בית): המנגנון מוכן — מוסיפים בדיקת קוד ב-`submitChapter` מול טאב קודים. לא נדרש שינוי מבני.

## פיתוח מקומי

פותחים את `index.html` ישירות בדפדפן — כשה-`SCRIPT_URL` לא מוגדר המערכת עובדת במצב מקומי מלא (שאלות מ-`survey-data.js`, שמירה ב-localStorage) — מצוין להתנסות ולבדיקת נוסחים לפני עלייה לאוויר.
